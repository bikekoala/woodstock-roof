# 决策记录 — 为什么是现在这样

> 记录关键选择及其理由，避免新 agent 重走弯路。按时间顺序。

## D1. 工具选择：自建 Three.js 网页，而非 Onshape/Zoo（落地阶段）

- **考虑过**：Onshape（参数化最强、免费、有标准件库）、Fusion 360（个人免费、带 McMaster 螺丝库）、Zoo/KCL（AI/代码 CAD）、厂商框架配置器（item/MISUMI，带真实螺丝库）。
- **结论**：先用**自建可交互 HTML（Three.js）** 做形态推敲——参数滑块联动、BOM、点击看详情，稳定、零安装、改起来快。**真正出可加工模型再转 Onshape。**
- **Zoo 已废弃**：v1.2.16 视口卡、KCL 语法迭代快（offsetPlane/startProfile 等签名不稳），复杂装配体它做不了。仅适合体验单零件。

## D2. 材料：4040 只用在承力主体，其余 2020

- 4040：底框、角立柱、顶框、底框中央纵梁（承重+抗扭+受弯）。
- 2020：托盘、中央竖撑、轻框。
- 1515 V-slot：伸缩滑轨（薄 + 配 V 轮，有现成生态；1010 太小无生态、悬臂会塌）。
- **理由**：SKU 越少越便宜越好装；把粗料只花在刀刃上。

## D3. 结构是「立方体箱体」，不是平托盘 ★关键

- 早期画成平框 + 悬空板，车主纠正：**应该是个立方体**（底框 + 4 立柱 + 顶框，12 根边），太阳能板/水箱是箱体内不同高度的"层架"。
- 立柱的意义 = 给各层提供固定点。这解答了反复出现的"太阳能板/水箱固定在哪"。

## D4. 水箱位置：选了「顶层」（方案 A），放弃「低位吊篮」（方案 B）

- **受力分析**：水箱(~30kg)放顶层 = 最差受力位（顶框被迫受弯、重心高）；放底层吊篮则载荷直传 4040 底框和车顶、重心低。
- 工程上 B 更优（低重心、更轻、更稳），但**车主选了 A**（贴近其原始构想：顶层水箱 + 顶板）。
- **代价（已知并接受）**：顶框/立柱须升 4040；总高 ~225mm 略超 20cm；重心偏高。
- 若日后想压高度/降重心，B 方案随时可回退（见 reference/旧定稿-v2.md 的低位思路）。

## D5. 加固：中央纵梁脊 + 中央竖撑，去掉斜撑

- 先加过顶部 4 角斜撑(B1)，但**斜撑会挡滑板/托盘** → 移除。
- 改为：**底框中央纵梁 M3** + **托盘中央纵梁 D4**（托住各横梁/横档中点）+ **中央竖撑 S1**（前后各一，从底框中纵梁撑到托盘中纵梁，从下托住中线）。
- **抗侧倾**改由角码刚度承担：8 个箱角 + 连车点用 **L 加强（带筋）角码**，普通接头用普通角码。
- D4 一度用 4040 太厚、戳到下面滑板 → 降为 2020、放回托盘平面。

## D6. 连接方式：T 槽 + L 角码 + T 螺母 + 螺栓

- 标准、免钻免焊，到家只用内六角。
- 关键接头（箱角、连车点、托盘吊柱接顶框）用**加强型铸铝角码**，别用最薄冲压件（动载环境）。
- 型材下单时备注"端面铣平、公差 ±0.2mm"，定长切割。

## D7. 太阳能板尺寸统一

- 板 A/B/C 统一 1000×1200（车主要求 PC 与另两张同尺寸）；顶板较小、固定。
- ⚠️ 2026-06-10 被 D9 部分取代：A/B 缩至 1000×1180 解滑出冲突，C 仍 1000×1200。

## D8. 实测回填：railSpacing=1020、railLength=1300（2026-06-10）

- 实测：OEM 纵梁宽 40；两根导轨**不平行**，中心距前端 1070 → 后端 1020 渐缩。车主定按窄端 **1020** 计。
- 导轨纵向构成：平直支撑段（两支撑点间）正好 1000，前下坡段 250、后下坡段 200 → 可用长度 1000~1450。车主折中取 **1300**，前后各悬挑出支撑点 ~150（4040 下挠度可忽略）。
- 渐缩的影响（按 D11 修正后的"总外宽"语义）：1020 取作箱体总外宽 = OEM 导轨外缘跨距；两纵梁 M1 各 40 居于两侧 → 梁中心距 980 = 导轨中心距（1020−40）。前端导轨外缘 1070 时中心距 1030，相对后端 980 偏 +25mm/侧分摊后约 ±稍许，落在 40 宽梁投影内 → U 卡扣可适配，装配以现场对位为准（实车渐缩未必线性）。
- zBot（导轨抬高）仍未实测。

## D9. 滑板 A/B 缩短居中：1000×(railLength−120)，解双冲突 ★

- AABB 全行程扫描发现**两处冲突**，比此前记录的更严重：
  ① **S1 竖撑**（z75–115）竖穿中层板高度（板B z83–91、板A z97–105）——**收起状态即互相穿插**，不只是滑动时撞；
  ② **角立柱 C1**：板长=箱长时，同侧立柱间净口仅 railLength−80，板四角全行程必撞——此前未发现。
- 解法：板 A/B 长边改 **railLength−120**（现 1180）、y 向居中 → 对立柱净隙 20、对 S1 净隙 40，全行程 51 位置扫描零碰撞。**S1 原位保留，结构件零改动。**
- 否决的替代：「竖撑避让行程」单独无解——板层横在 S1 必经高度、又扫满箱内全宽；且立柱冲突仍在，板照样出不去。「中间留缝」同理：缝随板动，挡不住扫掠路径。
- 代价：A/B 各损失 1000×20（相对 1300 箱体下可达的 1200），合计 0.04㎡，可忽略。

## D10. 实体干涉自检闸门：`scripts/check-interference.js` ★方法

- **动机**：车主指出"有的杆和立柱重叠了，你如何保证设计符合物理"。靠肉眼看 3D 漏判——这正是脚本模型相对 CAD 的短板。补法：把"无实体穿插"做成**自动测试**。
- 脚本从 `model.js` 取全部零件 AABB，对每一对做体积相交；滑板会动，故 deploy 0→1 全行程扫描取最坏穿深。分两类：
  - **非法穿插 (ILLEGAL)**：跨子系统/异平面实体互穿（托盘杆穿立柱、板插进骨架等）→ **必须 0，否则退出码 1**。
  - **接头重叠 (JOINT)**：同框构件在节点相交（角接/十字），装配切短拼接 → 仅提示。同框组：底框{M1,M2,M3}/顶框{T1,T2}/托盘{D1,D2,D3,D4}。端面贴合（穿深<0.5）自动豁免。
- **已挂进 pre-commit 钩子**：提交前先跑干涉检查，有非法穿插直接阻断（`--no-verify` 可强行跳过）。从此每次改 model.js 都被物理校验兜底。
- 局限（诚实记录）：只是 AABB 盒，不是真实 T 槽截面；接头是否真能切短对接、螺栓干涉、动态变形都要等转 Onshape 才能严判。它保证的是"没有两块实体占同一空间"，不等于"可制造性全过"。

## D11. 宽度语义纠错 + 托盘内缩：解 8 处穿柱 + 横梁虚长 ★

- 车主纠错两点，均已修复，干涉脚本复核非法穿插 8→0：
  1. **宽度重复计算**：旧代码把 `railSpacing` 当"内腔净宽"又外扩 `+2×40`，总外宽虚胖到 1100、横梁 M2/T2 多算 80。实为 **railSpacing = 箱体总外宽 = 1020**（OEM 导轨外缘跨距），两侧纵梁各占 40，横梁/内腔净宽 = 1020−80 = **940**。改：`outerW = railSpacing; gap = railSpacing − 2×40`。
  2. **托盘穿立柱**：托盘边梁 D2、横档 D3 旧为满外宽（D2 在 x=0/outerW−20、D3 长=outerW），四角各怼进角立柱 C1 内 20mm（8 处穿插）。改：边梁内缩贴立柱内侧（x=40 / outerW−60），横档在两边梁间对接（长=outerW−120=900），水箱同步缩到托盘内净宽 900。
- 连带：箱体 1020×1300×225、展开宽 3020；铝型材合计 21.66→20.26m（箱变窄）。板 A/B 保持 1000×1180，对立柱仍 20 净隙（现由 y 向让位实现，因立柱只在四角）。
- 残留 JOINT（非错，待 Onshape 精修切料）：M2×M3（底框中脊穿端横梁）、D1×D2（吊柱根接边梁）、D3×D4（托盘十字）。

## D12. 结构减脂：M3→2020、加中央吊撑 S2、T2→2020（2026-06-10）★

- 车主问"哪些杆能换细的"。按"是否在水箱承重路上"逐根判：水箱载荷两条路——①上吊 托盘→D1→顶框 T1/T2→立柱 C1→底框→导轨；②下撑 托盘→S1→M2→M1→导轨。在路上的粗、不在的细。
- **M3（底框中脊）→ 2020**：两条路都不在它身上，只作中脊/抗扭。降 2020（顶面齐平 z55–75），可省可留；保留作板C/底层滑轨的安装脊。
- **T1 留 4040**：托盘正吊在其跨中，头号受弯件。挠度估算 4040≈0.7mm vs 2020≈8.9mm（满水 ~40kg、半数走上吊路）。动它=推翻方案A顶框升级（D4）。
- **C1 留 4040**：受压应力仅 0.06MPa（可忽略），但它是去斜撑后(D5)唯一的抗侧倾竖骨；且总长仅 0.44m，换细省不了料却削弱抗摇。
- **加中央吊撑 S2 + T2→2020**：S2（2020）从顶框中横梁 T2 竖到托盘中脊 D4（D4 被 S1 撑着、刚度硬），等于给 T2 跨中加支点、跨度减半 → T2 挠度 3.2→0.4mm，遂可降 2020。S2 位于正中 (x510,y650)，水箱正后方，不碰水箱/滑板（干涉自检过）。
- **效果**：4040 仅剩 M1/M2/C1/T1（7.52m），M3/T2 转 2020；结构净轻约 6.5kg（顶部减重，利于重心）。非法穿插仍 0。
- 注：S2 价值不在"扛"，在"解锁 T2 降级"——四点菱形吊撑(D1×2 + S1×2)本身已稳定，正中并非必需支点。

## D13. D1 吊柱对齐 T1（修悬空）+ 自检加"承接检查"（2026-06-10）★

- 车主发现 **D1 没挂在 T1 上**。核实属实，且是两步叠加的锅：
  - D11 把托盘内缩，D1 随边梁 D2 落到 x40–60（在 T1 x0–40 内侧 20mm，本就只剩棱边接触）；
  - D12 为做平顶面把 T2 抬到 z205–225，**把 D1 顶上原本接的 T2 抬走**，D1 顶端 z185 与 T2 底 z205 之间留 20mm，彻底悬空。
- **干涉自检漏报**：它只查"两块实体重叠"，查不出"该接没接上"的缝隙（D10 已记此局限，被车主一眼抓出）。
- 修：D1 移到 **x20–40 / x980–1000**，对齐顶框纵梁 T1 内半，整 20×20 顶面贴 T1 底面承接（不再是棱边/悬空）。底端与边梁 D2 并接（角码）。
- **自检增强**：新增"承接检查"——竖向承力件（C1/S1/S2/D1）的承力端必须与被接构件有**面积投影重叠**（仅棱边/缝隙判为悬空），顶端必查、底端按是否"坐落"查（D1 底为角码并接不查）。回归测试：把 D1 改回旧位，立即报"两根吊柱顶端悬空"。已纳入 pre-commit 阻断条件。
- **关于"两边是否要再加两根竖撑"**（车主问）：不需要——左右两侧的吊撑**就是 D1**。中线 yMid 上本是 `D1左·S2中·D1右` 三根对称挂顶框，此前只因 D1 没挂对才显得缺。真正欠支撑的是**水箱前两角**（悬臂估算 ~15mm 下垂），若加料应加在前部托住水箱，而非中侧——待定。

## D14. 托盘四角对接角立柱 C1（直传 M1/导轨）；保留 D1（2026-06-10）★

- 车主两问：① 前后要不要补支撑；② D2/D3 不应该直接接到 M1 上吗，让我把模型捋清。
- 逐层梳理（z，mm）：板C下挂 21–29 ┃ 底框 M1/M2 35–75（坐导轨）┃ **中层滑板 A/B 83–105（满宽 x10–1010）** ┃ 托盘 D 115–135 ┃ 水箱 135–185 ┃ 顶框 T1 185–225。
- **答②（关键几何约束）**：托盘(z115) 想竖直下接 M1(z35–75)，中段两侧必穿过**滑板层 z83–105**（满宽），下杆会挡死滑板行程 → 中段两侧只能往上吊（D1 挂 T1）。但"接 M1"在能接处**已做到**：角立柱 **C1 底面正坐 M1 顶(z75=z75)**，托盘四角经 D2↔C1 → C1 → **M1 → 导轨直传**；中线前后由 S1 落底框。挡不住的仅"中段两侧"，故必须 D1 上吊。
- **改动**：D11 内缩后 D2 边梁内侧面 x=40/980 已正贴 C1 内侧面、两端(y0–40/1260–1300)同处前后立柱、z 在立柱高度内 → **D2 两端用加强角码对接 C1**（无新增型材），托盘四角坐落立柱、兜住水箱前两角（消 ~15mm 下垂）。
- **答①**：前后中线已有 S1前/S2中/S1后，不缺；欠的是四角，由本条 D2↔C1 解决。
- **保留 D1**：曾试删 D1（误以为冗余）→ 中段两侧失托、托盘跨度变大受力增（车主纠正），已恢复。承托最终 = 四角坐 C1 + 中段两侧 D1 挂 T1 + 中心 S1/S2。
- 自检：干涉 0 / 承接 0；型材 20.33m。
- ⚠️ 2026-06-10 被 D15 取代接法：D14 的"细 D2 + 四角角码"被车主否，改为"D2 升 4040 直接怼进 C1"。

## D15. 托盘边梁 D2 升 4040、与 M1/C1 同规格、两端直接怼进角立柱 C1 ★（2026-06-10）

- 改动：
  - **D2 → 4040**（40×40），左右各落在 **x0–40 / x980–1020**（与 M1、C1 完全同 x、同宽，竖直叠成一条 M1→C1→D2 的边柱系）。
  - D2 长 = railLength−2×40 = **1220**，在前后两立柱之间对接（y40↔1260 正贴立柱内面，零缝零穿）。
  - **取消 D1 吊柱**：D2 升 4040 后两端怼立柱、自身够刚（跨中垂≈1mm），中段吊柱由立柱顶替而冗余，且会与粗 D2 重叠。**与 D14 误删不同**——那时 D2 还是细 2020、必须 D1 兜；现在 D2 自己扛住。

## D16. 前后端中央立柱 S3：水平拦阻不是力学件（2026-06-10）★

- 车主问"S1 上面是不是也应该有立柱"，agent 先按力学回答"不必要"——**漏判**：S1 顶到托盘就停，前后端中线 D3(z135) 与 T2(z205) 之间留有 70mm 高的开口；水箱(z115–185) 上半段 50mm 完全裸露，急刹/颠簸时整箱水(~30kg) 会从前端口飞出，后端置物区同理无拦阻。车主纠正："除了力学，还要考虑物理与生活经验。"
- **教训**（写进 CLAUDE.md §9）：**结构件的功能 = 承力 + 拦阻 + 约束 + 维护可达**。下次评估改动主动过一遍这层，别只算挠度。
- 改动：新增 **S3 前后端中央立柱**（2020 / 70mm / ×2），从 D4 端部 → T2 端横梁中点，几何与 S2 镜像（前端 y=0、后端 y=railLength−20）。底面贴 D4 顶面（z135）承接，顶面贴 T2 底面（z205）承接。
- 功能：① 把前后端中线封成"门框"，水箱与置物被骨架包住；② 作为绑带/网兜/挡板的锚点（行驶前可绕水箱前后系一道带子）。
- **非力学件**：T1 跨中无支撑这件事 S3 改变不了；前后端 T2 本来载荷就小（顶板边缘 ~2kg，跨 940 2020 挠度 ~0.5mm），S3 给它加跨中支点是附带福利不是动机。所以 **T2 仍保 2020 不升 4040**——升 4040 多 1.2kg 在顶部、重心反向，不值。
- 自检：干涉 0 / 承接 0；型材 20.23→**20.37m**（+0.14m 2020）；BOM 件数 30→32。S3 的承接被加进 check-interference.js 的 TOP/BOT_BEARING 列表，回归保护。
