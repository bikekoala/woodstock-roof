# 决策记录 — 为什么是现在这样

> 记录关键选择及其理由，避免新 agent 重走弯路。按时间顺序。

## D1. 工具选择：自建 Three.js 网页，而非 Onshape/Zoo（落地阶段）

- **考虑过**：Onshape（参数化最强、免费、有标准件库）、Fusion 360（个人免费、带 McMaster 螺丝库）、Zoo/KCL（AI/代码 CAD）、厂商框架配置器（item/MISUMI，带真实螺丝库）。
- **结论**：先用**自建可交互 HTML（Three.js）** 做形态推敲——参数滑块联动、BOM、点击看详情，稳定、零安装、改起来快。**真正出可加工模型再转 Onshape。**
- **Zoo 已废弃**：v1.2.16 视口卡、KCL 语法迭代快（offsetPlane/startProfile 等签名不稳），复杂装配体它做不了。仅适合体验单零件。

## D2. 材料：4040 只用在承力主体，其余 2020

- 4040：底框、角立柱、顶框、底框中央纵梁（承重+抗扭+受弯）。
- 2020：托盘、中央竖撑、轻框。
- 1515 V-slot：伸缩滑轨（薄 + 配 V 轮，有现成生态；1010 太小无生态、悬臂会塌）。
- **理由**：SKU 越少越便宜越好装；把粗料只花在刀刃上。

## D3. 结构是「立方体箱体」，不是平托盘 ★关键

- 早期画成平框 + 悬空板，车主纠正：**应该是个立方体**（底框 + 4 立柱 + 顶框，12 根边），太阳能板/水箱是箱体内不同高度的"层架"。
- 立柱的意义 = 给各层提供固定点。这解答了反复出现的"太阳能板/水箱固定在哪"。

## D4. 水箱位置：选了「顶层」（方案 A），放弃「低位吊篮」（方案 B）

- **受力分析**：水箱(~30kg)放顶层 = 最差受力位（顶框被迫受弯、重心高）；放底层吊篮则载荷直传 4040 底框和车顶、重心低。
- 工程上 B 更优（低重心、更轻、更稳），但**车主选了 A**（贴近其原始构想：顶层水箱 + 顶板）。
- **代价（已知并接受）**：顶框/立柱须升 4040；总高 ~225mm 略超 20cm；重心偏高。
- 若日后想压高度/降重心，B 方案随时可回退（见 reference/旧定稿-v2.md 的低位思路）。

## D5. 加固：中央纵梁脊 + 中央竖撑，去掉斜撑

- 先加过顶部 4 角斜撑(B1)，但**斜撑会挡滑板/托盘** → 移除。
- 改为：**底框中央纵梁 M3** + **托盘中央纵梁 D4**（托住各横梁/横档中点）+ **中央竖撑 S1**（前后各一，从底框中纵梁撑到托盘中纵梁，从下托住中线）。
- **抗侧倾**改由角码刚度承担：8 个箱角 + 连车点用 **L 加强（带筋）角码**，普通接头用普通角码。
- D4 一度用 4040 太厚、戳到下面滑板 → 降为 2020、放回托盘平面。

## D6. 连接方式：T 槽 + L 角码 + T 螺母 + 螺栓

- 标准、免钻免焊，到家只用内六角。
- 关键接头（箱角、连车点、托盘吊柱接顶框）用**加强型铸铝角码**，别用最薄冲压件（动载环境）。
- 型材下单时备注"端面铣平、公差 ±0.2mm"，定长切割。

## D7. 太阳能板尺寸统一

- 板 A/B/C 统一 1000×1200（车主要求 PC 与另两张同尺寸）；顶板较小、固定。
