# CLAUDE.md — woodstock-roof 项目上手总纲

> 给接手本项目的 AI agent / 协作者：读完本文即可了解全部来龙去脉。
> 最后更新：2026-06-10

## 1. 这是什么项目

为 **荣威 eRX5 2017 PHEV（白色）** 设计并 DIY 组装一套**车顶多层伸缩太阳能平台**。核心诉求：
- 用**标准铝型材**搭建（淘宝定长切割，到家只用内六角扳手组装，零钻零焊）
- 集成可电动**伸缩展开**的柔性太阳能板、扁平**水箱**、开放**置物区**
- 整体尽量低矮、受力合理、重心可控
- 与车主既有的 **Woodstock ESP32 项目**复用控制逻辑

车主不是机械专业，靠 3D 预览反复对齐脑中构想。**沟通要点**：他对结构细节很敏感、会逐处纠错；先理解物理结构再动手，别凭空堆几何。

## 3. 设计概要（当前方案 A）

**一个铝型材立方体箱体，竖直方向分层利用。** 侧视：

```
顶层前半：顶板(固定太阳能) 盖在 ┊ 顶层后半：置物区(开放, DIY)
          水箱 上方
          ╔════ 顶框 4040 (T1×2 + T2×3含中梁) ════╗
          ║  [整体吊托盘 2020: D1吊柱/D2边梁/      ║
          ║   D3横档×5/D4中央纵梁]  ← 兜水箱        ║   立柱 C1×4 (4040)
          ║  板A(左滑) / 板B(右滑) 叠放(中层)       ║
          ╚════ 底框 4040 (M1×2+M2×2+M3中央纵梁) ══╝  坐在车顶 OEM 导轨上
              板C 下挂(前后滑出)
          S1 中央竖撑×2: 底框中纵梁→托盘中纵梁(从下托住中线)
```

要点：
- **骨架** = 底框 + 4 角立柱 + 顶框，全 4040，是受力主体。
- **托盘** 从顶框往下吊（D1/D2），整体连成一个框，前半装水箱、后半置物。
- **太阳能板 4 块**：A/B 中层叠放左右滑出，C 底框下挂前后滑出，顶板固定盖顶框前半。均 ETFE 柔性板，1000×1200（顶板较小）。
- **中央纵梁脊**（M3 底、D4 托盘）+ **中央竖撑 S1** 托住跨度中点，替代斜撑。
- **连接**：T 槽 + L 型加强角码 + T 螺母 + 螺栓。

### 数据自动同步（重要）
- **唯一数据源 = `cad/model.js`**。零件的几何与 BOM 都从它来。
- 改零件 → 改 `model.js` → HTML 刷新即更新；`docs/bom.md` 由 `node scripts/gen-bom.js` 重算（**git commit 时 pre-commit 钩子自动执行并加入提交**）。
- 调过参数想让 BOM 反映：把新值写进 `model.js` 的 `DEFAULTS`（滑块改的是运行时，不落盘）。
- **每次改了设计/model.js，agent 应同步更新 design-spec.md / decisions.md / CLAUDE.md 相关处**，别让文档漂移。

## 5. 主控参数（在 HTML 顶部 `const P`）

改这些数字，整个模型联动重建。**实测真车后回填。**

| 参数 | 含义 | 默认 | 备注 |
|---|---|---|---|
| `railSpacing` | OEM 导轨中心距 | 1000 | **必须实测！** |
| `railLength` | 主纵梁长 | 1200 | |
| `boxHeight` | 箱体内净高(立柱高) | 110 | 调小可压总高 |
| `panelTravel` | 单侧抽出行程 | 1000 | |
| `tankH` | 水箱/托盘深 | 70 | 定制扁箱 |
| 固定 | 4040/2020、板1000×1200、zBot=35(OEM抬高) | | |

派生：外宽 = railSpacing+80 ≈ 1080；**总高 ≈ 225mm（略超 20cm 红线，方案A代价）**；展开总宽 ≈ railSpacing+80+2×panelTravel ≈ 3080（行驶前须收回）。

## 7. 编辑约定

- **改零件 = 改 `cad/model.js` 的 `buildModel()`**，不要在 HTML 里写零件定义。
- **坐标系**：X=左右(宽)，Y=前后(长)，Z=上。Three.js 是 Y-up，已在 HTML 的 `addBeam` 内做映射（设计 Z→three Y）。
- **零件记录字段**：`{vis, code,name,profile,section,length,cat, xw,yd,zh, x,y,z, color, opacity}`。`vis`=控制显隐的开关名（null 常显），按最小角点 `(x,y,z)` 定位。`cat ∈ struct/tray/panel/tank`。
- HTML 的 `build()` 只是遍历 `RoofModel.buildModel(P)` 调 `addBeam` 渲染；BOM 同源。
- 改完：① 跑 `node scripts/gen-bom.js`（或直接 commit）同步 BOM；② 让车主**刷新浏览器(Cmd+R)** 看 3D。
