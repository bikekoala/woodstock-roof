## 1. 这是什么项目

为 **荣威 eRX5 2017 PHEV（白色）** 设计并 DIY 组装一套**车顶多层伸缩太阳能平台**。核心诉求：
- 用**标准铝型材**搭建（淘宝定长切割，到家只用内六角扳手组装，零钻零焊）
- 集成可电动**伸缩展开**的柔性太阳能板、扁平**水箱**、开放**置物区**
- 整体尽量低矮、受力合理、重心可控
- 与车主既有的 **Woodstock ESP32 项目**复用控制逻辑

车主不是机械专业，靠 3D 预览反复对齐脑中构想。**沟通要点**：他对结构细节很敏感、会逐处纠错；先理解物理结构再动手，别凭空堆几何。

## 3. 设计概要（当前方案 A）

**一个铝型材立方体箱体，竖直方向分层利用。** 侧视：

```
顶层前半：顶板(固定太阳能) 盖在 ┊ 顶层后半：置物区(开放, DIY)
          水箱 上方
          ╔═ 顶框 (T1×2纵梁 4040 + T2×3横梁 2020) ═╗
          ║  [托盘: D2边梁 4040(怼进立柱)/          ║
          ║   D3横档×5 2020/D4中脊 2020]  兜水箱     ║   立柱 C1×4 (4040)
          ║  板A(左滑) / 板B(右滑) 叠放(中层)       ║
          ╚═ 底框 (M1×2+M2×2 用4040 + M3中脊用2020) ═╝  坐在车顶 OEM 导轨上
              板C 下挂(前后滑出)
          S1 中央竖撑×2(下:底框→托盘) + S2 中央吊撑×1(上:托盘→顶框) 托住中线
```

要点：
- **骨架** = 底框 + 4 角立柱 + 顶框。承力主梁 M1/M2/C1/T1 + **托盘边梁 D2** 用 4040；中脊 M3、顶框横梁 T2、竖撑 S1/S2、托盘横档 D3/中脊 D4 用 2020。
- **托盘承托**（见 D15）：边梁 **D2 升 4040，与 M1/C1 同宽同 x（x0–40/980–1020）、两端直接怼进角立柱 C1** → 载荷 D2→C1→M1→导轨直传；中心 S1/S2 托中线。D2 够刚（跨中垂~1mm）故无中段吊柱。前半装水箱、后半置物。
- **太阳能板 4 块**：A/B 中层叠放左右滑出，C 底框下挂前后滑出，顶板固定盖顶框前半。均 ETFE 柔性板：C 1000×1200，A/B 1000×1180（=railLength−120，居中让开立柱与 S1），顶板较小。
- **中央纵梁脊**（M3 底、D4 托盘）+ **中央竖撑 S1** 托住跨度中点，替代斜撑。
- **连接**：T 槽 + L 型加强角码 + T 螺母 + 螺栓。

### 数据自动同步（重要）
- **唯一数据源 = `cad/model.js`**。零件的几何与 BOM 都从它来。
- 改零件 → 改 `model.js` → HTML 刷新即更新；`docs/bom.md` 由 `node scripts/gen-bom.js` 重算（**git commit 时 pre-commit 钩子自动执行并加入提交**）。
- 调过参数想让 BOM 反映：把新值写进 `model.js` 的 `DEFAULTS`（滑块改的是运行时，不落盘）。
- **每次改了设计/model.js，agent 应同步更新 design-spec.md / decisions.md / CLAUDE.md 相关处**，别让文档漂移。

## 5. 主控参数（model.js 的 `DEFAULTS`，HTML 启动时引用）

改这些数字，整个模型联动重建。

| 参数 | 含义 | 默认 | 备注 |
|---|---|---|---|
| `railSpacing` | **箱体总外宽**（=导轨外缘跨距） | 1020 | 已实测后端，前端 1070 渐缩；两侧纵梁各 40 已含在内，横梁净宽=1020−80=940 |
| `railLength` | 主纵梁长 | 1300 | 已实测折中：支撑段 1000+前坡 250/后坡 200 |
| `boxHeight` | 箱体内净高(立柱高) | 110 | 调小可压总高 |
| `panelTravel` | 单侧抽出行程 | 1000 | |
| `tankH` | 水箱/托盘深 | 70 | 定制扁箱 |
| 固定 | 4040/2020、板C 1000×1200、板A/B=railLength−120、zBot=35(OEM抬高，**待实测**) | | |

派生：总外宽 = railSpacing = 1020（含两侧纵梁各 40）；内腔净宽 = 1020−80 = 940；**总高 ≈ 225mm（略超 20cm 红线，方案A代价）**；展开总宽 = 1020+2×panelTravel = 3020（行驶前须收回）；板 A/B 居中后对角立柱净隙 20、对 S1 净隙 40；托盘内缩至立柱内侧（横档/水箱 900 宽）。

## 7. 编辑约定

- **改零件 = 改 `cad/model.js` 的 `buildModel()`**，不要在 HTML 里写零件定义。
- **坐标系**：X=左右(宽)，Y=前后(长)，Z=上。Three.js 是 Y-up，已在 HTML 的 `addBeam` 内做映射（设计 Z→three Y）。
- **零件记录字段**：`{vis, code,name,profile,section,length,cat, xw,yd,zh, x,y,z, color, opacity}`。`vis`=控制显隐的开关名（null 常显），按最小角点 `(x,y,z)` 定位。`cat ∈ struct/tray/panel/tank`。
- HTML 的 `build()` 只是遍历 `RoofModel.buildModel(P)` 调 `addBeam` 渲染；BOM 同源。
- 改完：① 跑 `node scripts/check-interference.js` 确认非法穿插=0；② 跑 `node scripts/gen-bom.js`（或直接 commit）同步 BOM；③ 让车主**刷新浏览器(Cmd+R)** 看 3D。
- **坐标/尺寸约定**：`railSpacing` = 箱体总外宽（非内距）；零件按最小角点 `(x,y,z)` + 尺寸 `(xw,yd,zh)` 定位，加新零件后必须过干涉自检。

## 9. 沟通与协作风格

- 车主用中文，喜欢探讨式（"探讨下""你觉得呢"）——给判断+理由，不要只堆选项。
- 涉及受力/结构，给**具体数字**（如挠度估算）他很买账。
- 改 3D 后一句话说清"改了什么 + 让他刷新 + 请他验证哪几点"。
- 别怕承认理解错——本项目前期来回纠正多次，坦诚高效。
- **评估构件别只算力学**：**结构件功能 = 承力 + 拦阻(惯性/外溢) + 约束(绑带锚点/挡板) + 维护可达**。车主从生活经验角度纠错过多次（如 D16：S3 立柱不是承力件，是防水箱刹车飞出）；同类还要主动想：排水、风载/雨水反溅、装拆顺序、手能否伸进维护、柔性板高温变形等。
